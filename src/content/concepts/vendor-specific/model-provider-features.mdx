> **Stub** — planned territory on the atlas, deliberately at the rim. This concept will cover:
> how each provider wraps the shared primitives (tokens, tool calling, structured output,
> caching) in its own proprietary surface — a specific JSON-mode, a prompt-caching API, hosted
> tools, batch endpoints, provider-side memory — the genuine convenience these buy, and the
> honest cost: they are the fastest-moving, least-portable layer, so building on them is a
> lock-in decision to make on purpose. The rule the whole atlas argues: learn the primitive
> first (it's stable), treat the vendor's wrapper as a convenience you can swap. Builds on
> Tool calling and Structured outputs.
